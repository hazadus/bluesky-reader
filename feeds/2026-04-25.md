💬 Python Tip #115 (of 365):

Instead of manually caching computed attribute values, use functools.cached_property. 🧵

#Python #DailyPythonTip

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 A @property recomputes its value every time it's accessed.

That's fine for computing values that should stay synced each time the attribute is accessed.

But if the computation is expensive, you might not want to redo it every time.

https://pym.dev/p/24n3p/

class FileStats:

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 But I would recommend using functools.cached_property instead.

With @cached_property the value is computed once, then cached as a regular attribute.

The first access does the work. Every access after that is instant.

https://pym.dev/p/2mjp9/

![from functools import cached_property
from pathlib import Path

class FileStats:
    def __init__(self, filename):
        self.path = Path(filename)

    @cached_property
    def line_count(self):
        with open(self.path) as file:
            return sum(1 for line in file)](https://cdn.bsky.app/img/feed_fullsize/plain/did:plc:nanrhx7h54a4p72udpmptgop/bafkreicecpjtxaqvxgixj3g7ihezyolpcrd2zddhhizmhsddt3e2462wfm)
