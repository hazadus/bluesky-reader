**🕒 21:19 👤 @trey.io (Trey Hunner)**
💬 Python Tip #182 (of 365):

Use textwrap.dedent() to unindent multi-line strings. 🧵

A multi-line string inside an indented function inherits that indentation.

textwrap.dedent() strips the common leading whitespace so your output looks right.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mpmh2fqj7k2w)

----
**🕒 21:19 👤 @trey.io (Trey Hunner)**
💬 Without dedent(), the code's indentation ends up inside the string:

>>> def help_text():
...     return """
...         Usage: my_tool [options]
...         -h, --help  Show this message
...     """
...
>>> print(help_text())

        Usage: my_tool [options]
        -h, --help  Show this message

[Открыть пост](https://bsky.app/profile/trey.io/post/3mpmh2fquwk2w)

----
**🕒 21:19 👤 @trey.io (Trey Hunner)**
💬 With textwrap.dedent(), the string can stay indented in your code but the result is clean:

>>> from textwrap import dedent
>>> print(dedent(help_text()))

Usage: my_tool [options]
-h, --help  Show this message

dedent() removes the common leading whitespace from every line!

[Открыть пост](https://bsky.app/profile/trey.io/post/3mpmh2fqvvs2w)

----
**🕒 21:19 👤 @trey.io (Trey Hunner)**
💬 I often define a small helper that also removes the leading newline (which comes from starting the string on its own line):

>>> def undent(text):
...     return dedent(text).removeprefix("\n")
...
>>> print(undent(help_text()))
Usage: my_tool [options]
-h, --help  Show this message

[Открыть пост](https://bsky.app/profile/trey.io/post/3mpmh2fqwv22w)

**🕒 21:19 👤 @trey.io (Trey Hunner)**

I DON'T recommend using cleandoc() unless you're working with docstrings. Its name will cause more confusion than it's worth.
