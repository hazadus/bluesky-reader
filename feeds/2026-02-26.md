ğŸ’¬ Python Tip #57 (of 365):

Invent your own comprehensions ğŸ§µ

You can invent your own comprehensions in Python... well, sort of.

You can at least invent "I can't believe it's not a comprehension" comprehensions.

#Python #DailyPythonTip

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mfrv6hasgo2y)

----
**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ Python has set comprehensions:

word_set = {w.strip(".,!?\"'") for w in text.split()}

But it doesn't have frozenset comprehensions.

There is a built-in frozenset type, but there's no such thing as a frozenset comprehension!

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mfrv6hk4sa2n)

----
**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ But the frozenset class will accept an iterable of values to make into a new frozenset.

So we can (sort of) INVENT OUR OWN FROZENSET COMPREHENSION by passing a generator expression to frozenset:

word_set = frozenset(w.strip(".,!?\"'") for w in text.split())

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mfrv6hnc4e2o)

----
**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ Python's generator expressions can act SORT OF like an invent-your-own-comprehension tool.

Passing a generator expression into an iterable-accepting function/class makes something that looks and acts like a CUSTOM COMPREHENSION.

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mfrv6hpqpk2u)

----
**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ For example, this looks like a "tuple comprehension":

And this looks like a Counter comprehension:

words = Counter(w.strip(".,!?\"'") for w in text.split())

These are not REAL comprehensions, but they may as well be.

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mfrv6htjd227)

----
**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ Any class that accepts an iterable can accept a generator. This fact allows us to SORT OF invent our own comprehensions.

They're not REALLY custom comprehensions but they can serve the same purpose.

Custom collections are comprehension-like tools waiting to happen!
