**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #158 (of 365):

If you had to choose, prefer automated tests over type annotations. 🧵

Of course... you don't need to choose between these two. You can have both!

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnpqwi6tzu2i)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 A type checker can tell you your function returns the wrong type but it can't tell you that it returns the wrong answer.

Type annotations catch a specific category of bug: passing the wrong KIND of object.

Tests can catch that and more: logic bugs, incorrect output, mishandled edge cases, etc.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnpqwif4bm2y)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Of course, it's not really either/or. Most projects that use type annotations also have a solid automated test suite along with annotations that can make a large codebase easier to navigate.

But if I needed to pick between these two, I'd always pick automated tests.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnpqwiheqk2w)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Personally, I lean toward duck typing plus a solid test suite and I often skip annotations.

When I DO use type annotations, I try to use them everywhere and I ALWAYS check them (see tip 152).

Whatever you decide about annotations, make sure you also write automated tests.
