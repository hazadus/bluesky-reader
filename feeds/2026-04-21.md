**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #111 (of 365):

Don't use the __class__ attribute. 🧵

To get the name of any object's class, you can use:

type(my_obj).__name__

You might occasionally see this instead:

my_obj.__class__.__name__

I don't recommend that approach.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mjzkyqqrd52r)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Directly accessing the __class__ attribute is a holdover Python 2.

In Python 2, the type(...) function didn't always return an object's class (thanks to the weird old-style vs new-style class distinction).

In Python 3, the type() function always returns the value of the __class__ attribute.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mjzkyqtlmn2p)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 But why not accessing the __class__ attribute directly?

Because when there's a higher-level alternative to direct dunder attribute/method access, we tend to prefer it.

Instead of x.__len__() and x.__str__(), we use len(x) and str(x). Both work, but the function calls are the higher-level approach.
