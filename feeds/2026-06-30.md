рЯТђ Python Tip #181 (of 365):

Don't use Python's "str.title()" method. рЯІµ

It capitalizes letters after apostrophes, turning "don't" into "Don'T".

#Python #DailyPythonTip

[–Ю—В–Ї—А—Л—В—М –њ–Њ—Б—В](https://bsky.app/profile/trey.io/post/3mpk3dykmyc23)

----
**рЯХТ 22:45 рЯС§ @trey.io (Trey Hunner)**
рЯТђ Here's the problem:

>>> "Python is a don't-miss language".title()
"Python Is A Don'T-Miss Language"

str.title() defines a "word" as anything following a non-letter character.
Since ' isn't a letter, the t in "don't" gets capitalized.

[–Ю—В–Ї—А—Л—В—М –њ–Њ—Б—В](https://bsky.app/profile/trey.io/post/3mpk3dyonhk2k)

----
**рЯХТ 22:45 рЯС§ @trey.io (Trey Hunner)**
рЯТђ For most cases, string.capwords is a better choice:

>>> import string
>>> string.capwords("Python is a don't-miss language")
"Python Is A Don't-Miss Language"

string.capwords splits on spaces and capitalizes only the first letter of each word, so contractions are handled correctly.

[–Ю—В–Ї—А—Л—В—М –њ–Њ—Б—В](https://bsky.app/profile/trey.io/post/3mpk3dyrak526)

рЯТђ string.capwords does miss words after hyphens though:

>>> string.capwords("step-by-step guide")
