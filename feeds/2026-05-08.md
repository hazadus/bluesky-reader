**🕒 18:09 👤 @trey.io (Trey Hunner)**
💬 Python Tip #128 (of 365):

Chain method calls for readability's sake 🧵

When an object's methods return the same type of object, prefer to chain calls together instead of juggling extra variables.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mled7ug2lw2o)

----
**🕒 18:09 👤 @trey.io (Trey Hunner)**
💬 String methods are the classic example of this.

Each one returns a new string, so they chain nicely:

normalized = message.strip().lower().replace(",", "")

Imagine if we made a "stripped" variable and a "lower_stripped" variable for those 2 intermediary steps. I don't think it'd help readability.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mled7ujvtr2k)

----
**🕒 18:09 👤 @trey.io (Trey Hunner)**
💬 pathlib.Path methods also return Path objects, so chain nicely too:

>>> from pathlib import Path
>>> path = Path("report.txt")
>>> path.with_stem(f"{path.stem}_final").with_suffix(".md")
PosixPath('report_final.md')

[Открыть пост](https://bsky.app/profile/trey.io/post/3mled7umub32c)

----
**🕒 18:09 👤 @trey.io (Trey Hunner)**
💬 Django's QuerySet objects are another example:

recent_logins = (
    User.objects
    .filter(is_active=True)
    .exclude(last_login=None)
    .order_by("-last_login")[:10]
)

Wrapping a long chain of method calls in parentheses lets us put each method call on its own line for readability's sake.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mled7up65b2q)

----
**🕒 18:09 👤 @trey.io (Trey Hunner)**
💬 Here's a (mostly) unchained version of that:

active_users = User.objects.filter(is_active=True)
with_last_login = active_users.exclude(last_login=None)
recent_logins = with_last_login.order_by("-last_login")[:10]

I find the chained version easier to read. The variable names here don't really help.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mled7urngl2i)

----
**🕒 18:09 👤 @trey.io (Trey Hunner)**
💬 Chaining isn't ALWAYS clearer.

If an intermediate value is meaningful on its own, or if the chain gets long enough to be hard to follow, pull pieces out into named variables.

But for short sequences of transformations, it's often the most readable option.
