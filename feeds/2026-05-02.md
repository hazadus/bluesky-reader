💬 Python Tip #122 (of 365):

Pass a string to sys.exit to show an error message while exiting 🧵

If you pass a string to sys.exit, the string will be printed to standard error and your program will exit with an exit code that indicates "error".

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mkva53yhwf2v)

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 This:

import sys

sys.exit("Error: config file is invalid")

Does the same thing as this:

import sys

print("Error: config file is invalid", file=sys.stderr)
sys.exit(1)

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 To terminals, a zero exit code means "success" and a non-zero exit code means "error".

Most erroneous program exits simply exit with the code "1".

Unless you need to use a custom exit code for some reason, I'd consider passing strings to the exit function when you need to exit with an error.

💬 Sometimes you'll see a main function that returns its error code with a "sys.exit(main())" call in the program. In that case, errors are printed in the main function and numeric numeric exit codes are often returned ("return 0" or "return 1")

But even in that case, error strings can be used...

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 "return" will return None and "sys.exit(None)" is equivalent to "sys.exit(0)" (and to "sys.exit()").

So a bare "return" won't print an error message on exit but returning a string will.

More on using exiting Python processes: https://pym.dev/exiting-a-python-program/
