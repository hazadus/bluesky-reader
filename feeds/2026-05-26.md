**🕒 20:22 👤 @trey.io (Trey Hunner)**
💬 Python Tip #146 (of 365):

Don't use sorted to find the largest/smallest item(s) 🧵

Need the largest/smallest ones in an iterable?

>>> numbers = [199, 123, 29, 123, 18, 76, 76, 351]

Unless you need every item in sorted order, you probably don't need to sort your iterable.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mmrt2tnf5k25)

----
**🕒 20:22 👤 @trey.io (Trey Hunner)**
💬 You can use the max function:

>>> max(numbers)
351

To get the smallest value, you can use min:

>>> min(numbers)
18

Need the smallest/largest 4 items? Use nsmallest or nlargest from Python's heapq module:

>>> nsmallest(4, numbers)
[11, 18, 29, 47]
>>> nlargest(4, numbers)
[351, 275, 199, 123]

**🕒 20:22 👤 @trey.io (Trey Hunner)**
💬 Also note that all 4 of these functions can accept a key function, just like sorted can.
