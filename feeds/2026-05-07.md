**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Python Tip #127 (of 365):

Don't mutate objects passed in to your function, unless callers expect you to 🧵

Mutating an argument will change the original object that was passed in, which is likely to upset callers of your function.
#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mlcc62ldhy2p)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Passing an argument to a function is a form of assignment.

So when you mutate an argument, you're mutating the SAME object the caller has a reference to.

For example:

>>> def square_all(numbers):
...     for i, n in enumerate(numbers):
...         numbers[i] = n**2
...     return numbers

[Открыть пост](https://bsky.app/profile/trey.io/post/3mlcc63aop22r)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 When we call that square_all function with a "nums" list, the original list will change!

>>> nums = [2, 1, 3, 4, 7]
>>> square_all(nums)
[4, 1, 9, 16, 49]
>>> nums
[4, 1, 9, 16, 49]

That happened because square_all mutated the passed-in list object.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mlcc63cy722q)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Usually, it's better to return a NEW object instead of mutating the one passed in:

>>> def square_all(numbers):
...     return [n**2 for n in numbers]

Now the caller's list is left alone:

>>> nums = [2, 1, 3, 4, 7]
>>> square_all(nums)
[4, 1, 9, 16, 49]
>>> nums
[2, 1, 3, 4, 7]

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 There ARE times when mutation is the whole point: the list append method, the dictionary pop method, and many other methods are DESIGNED to mutate.

Don't mutate objects passed in to your function unless the caller expects you to.

More: https://pym.dev/pointers/#function-arguments-act-like-assignment-statements
