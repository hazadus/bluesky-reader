We use interval arithmetic on a high-fanout hierarchy of tiles (64³, 16³, 4³), followed by voxel and normal evaluation. At each stage of interval arithmetic, we compute a simplified tape for each tile containing only portions of the expression which are active.

After the root tile evaluation, tiles are sparse. Tiles and tapes use an atomic bump allocator to claim portions of a fixed buffer. The tile buffer is always sized to fit all possible tiles; the tape buffer can run out of space, in which case we fall back to the previous (unsimplified) tape.

Strata-sorted evaluation has a few advantages:

💬 Python Tip #130 (of 365):

Only use recursion for trees 🧵

Recursion is most useful for traversing or constructing TREE-LIKE STRUCTURES.

For most other problems, iteration is usually more readable.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mljdua5ibi2i)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Recursion is when a function calls itself.

Computer science classes almost always introduce recursion using factorial or fibonacci. Those are not great examples because those are both very clear (and fast) using a simple "for" loop instead.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mljduaa3jn2o)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Here's the recursive version of factorial:

def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n-1)

And here's the iterative version:

def factorial(n):
    result = 1
    for i in range(1, n+1):
        result *= i
    return result

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 If I were explaining "factorial" to a high schooler, I'd say "multiply all the numbers from 1 up to n." That's exactly what the iterative version does.

The recursive version ("n times factorial of n-1") is neat, but it's not any clearer.

I prefer using a "for" loop for factorial over recursion.

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Fibonacci is similar.

The recursive version:

def fibonacci(n):
    if n <= 1:
        return 1
    return fibonacci(n-1) + fibonacci(n-2)

And the iterative version:

def fibonacci(n):
    a = b = 1
    for _ in range(n-1):
        a, b = b, a+b
    return a
