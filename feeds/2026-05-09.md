**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #129 (of 365):

Use None as a sentinel value 🧵

None is great for representing nothingness, but it can also act as a placeholder for "no real value yet".

It's common to use None to distinguish "caller didn't pass an argument" from "caller passed a real value".

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mlgtfcypgf2e)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 A SENTINEL VALUE is a value whose presence carries special meaning: "stop here", "nothing seen yet", "caller didn't supply this". Code will specifically check for a sentinel to decide what to do next.

None is a great sentinel value, and Python itself uses None as a sentinel all over the place.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mlgtfd3sqr2p)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 For example:
• Passing None to str.split is the same as passing no separator.
• Passing "key=None" to sorted is the same as passing no key at all.

None isn't a PERFECT sentinel value... when None might legitimately show up as a real value in your data, you'll need a more unique sentinel value.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mlgtfd6dsj2p)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 But for MANY use cases, None is a perfectly fine sentinel value.

Note that Python 3.15 is adding custom sentinel values! If None might show up as a real value in your data, you might want to use a custom sentinel value instead. Or use object() in the meantime.

But that's for another's days tip...
