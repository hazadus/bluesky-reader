**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #139 (of 365):

Use "assert" statements for yourself but exceptions for your users 🧵

And "users" INCLUDES other Python programmers who are using your code function/class/etc.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mm7y3rrclp2k)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Need to ensure a condition is met before running a function? Start with assert:

assert len(character) == 1

If that case comes up as reasonably possible in real code, move from assert to "if" with a "raise".

if len(character) != 1:
    raise ValueError("'character' must be a string of length 1")

[Открыть пост](https://bsky.app/profile/trey.io/post/3mm7y3ryy4d2e)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Keep in mind that even before graduating to a proper exception, you can make "assert" a bit more descriptive by adding an error message:

assert len(character) == 1, "Character must be a string of length 1"

And note the lack of parens there: it's NOT a 2-item tuple (which would ALWAYS be truthy).
