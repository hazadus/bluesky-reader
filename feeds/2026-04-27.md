**🕒 23:42 👤 @trey.io (Trey Hunner)**
💬 Python Tip #117 (of 365):

Consider sys.argv before argparse 🧵

If you just need to accept a single argument to your CLI program, using sys.argv is simple:

try:
    [path] = sys.arg[1:]
except ValueError:
    print("Usage: program.‍py path", file=sys.stderr)

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mkjaoynumb2t)

----
**🕒 23:42 👤 @trey.io (Trey Hunner)**
💬 You can even handle --help:

if path == "--help":
    print("Usage: program.‍py path", file=sys.stderr)

Accepting any number of arguments with sys.argv is also pretty simple:

paths = sys.arg[1:]
if not paths or "--help" in paths:
    print("Usage: program.‍py paths [paths ...]", file=sys.stderr)

----
**🕒 23:42 👤 @trey.io (Trey Hunner)**
💬 If you want to do different things based on different numbers of arguments or you want to accept any --options or anything fancy, I'd switch to argparse.

But for the simple cases of no arguments, 1 argument, or unlimited arguments, consider just just using sys.argv.
