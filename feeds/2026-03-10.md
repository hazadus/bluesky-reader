**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #69 (of 365):

Joining and then printing? Consider using print() to do your joining instead. 🧵

print() can often function as an alternative to the string join method.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mgpxgooins2v)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Instead of passing an iterable to the string join and then passing the joined string to print:

print(", ".join(names))

You could unpack the iterable into a print call and specify the sep argument:

print(*names, sep=", ")

[Открыть пост](https://bsky.app/profile/trey.io/post/3mgpxgosgem2k)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 This has the added benefit that you can often remove string conversions (print converts the given values to strings but join doesn't).

So this:

print(", ".join(str(n) for n in numbers))

Can become this:

print(*numbers, sep=", ")
