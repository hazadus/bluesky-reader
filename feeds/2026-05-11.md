💬 Python Tip #131 (of 365):

Know the difference between iterables and iterators 🧵

New Python programmers sometimes say "iterator" when they mean "iterable".

These terms are NOT interchangeable.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mllue344ir2t)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Any objects that you can use a "for" loop to loop over is an iterable.

Iterators can be passed to the built-in next() function. Arbitrary iterables cannot be passed to next(). Only iterators can be.

But all iterables can be passed to iter() to get an iterator from them.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mllue374zo2v)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Also, generators are iterators.

So are files. And the return values of enumerate, zip, and reversed are all iterators.

You don't necessarily need to fully understand Python's iterator protocol to use iterators, but you should make sure you don't mix up the terms "iterable" and "iterator".
