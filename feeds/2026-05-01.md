**🕒 18:19 👤 @trey.io (Trey Hunner)**
💬 Python Tip #121 (of 365):

If your command-line script has a couple functions, put remaining logic in a main function 🧵

Python knows nothing about main functions but other Python developers do, which makes it a handy convention for making sense of code.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mksqjsbrru2v)

----
**🕒 18:19 👤 @trey.io (Trey Hunner)**
💬 Once your script grows a function or 2:

1. put all remaining logic in a main function
2. write a if __name__ == "__main__" conditional guard that calls main()
3. consider extracting argument-parsing logic into its own function: I often call this parse_args

[Открыть пост](https://bsky.app/profile/trey.io/post/3mksqjsdspx2c)

----
**🕒 18:19 👤 @trey.io (Trey Hunner)**
💬 Why use __name__ == "__main__"?

Well, in Python this is our way of asking "is our current module the entry point to our Python process?"

It asks "are we being run as a script (instead of imported as a module)?"

That check allows our script to be imported as a module OR be used as a CLI.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mksqjsfgxu2v)

----
**🕒 18:19 👤 @trey.io (Trey Hunner)**
💬 But there's a bigger reason to use __name__ == "__main__" and a main function: it's a VERY common convention.

If I spot a __name__ == "__main__" check, I know I'm in a script (not just a module).

If I see a main function, I assume it's the entry point that kicks off the script.
