**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #156 (of 365):

Parsing JSON with json.loads hands you a blob of nested dicts and lists with no structure. If you want structure, describe the shape. 🧵

A type checker (or pydantic) can then catch the typos and wrong assumptions for you.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnkpyfzabc2h)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Out of the box, parsed JSON has no schema. Nothing catches a mistyped key:

>>> import json
>>> book = json.loads(response)
>>> book["titel"]  # typo, no warning at all

You only find out at runtime, with a KeyError (if you're lucky) or a subtle bug (if you're not).

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnkpyg4xiz27)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 If you're annotating, describe the shape with a TypedDict:

from typing import TypedDict

class Book(TypedDict):
    title: str
    author: str
    year: int

book: Book = json.loads(response)

Note that this is STATIC only: json.loads still hands back a plain dict and does no validation at runtime.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnkpyg7npj2u)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 For RUNTIME validation, reach for a library like pydantic:

from pydantic import BaseModel

class Book(BaseModel):
    title: str
    author: str
    year: int

book = Book.model_validate_json(response)

Now bad data raises a clear error up front and you can use attributes instead of dict keys.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnkpygci6h2d)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Plain dictionaries work.

But if you're using type annotations to validate other parts of your code, you should really use a TypedDict to type check your JSON dictionaries also.

And if you're looking for actual VALIDATION of real data at runtime, use a library like pydantic.
