**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Python Tip #109 (of 365):

Don't overuse parentheses. 🧵

In Python, grouping parentheses can be wrapped around just about any expression. So it's easy to accidentally use parentheses where they're not needed.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mjuzrj4rtw2k)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 The parentheses in the condition of this "if" statement are unnecessary:

if (language == "Python"):
    print("It's Python!")

Unlike C or Java, Python's "if" doesn't need parentheses around conditions in Python:

if language == "Python":
    print("It's Python!")

[Открыть пост](https://bsky.app/profile/trey.io/post/3mjuzrj7uod2o)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 The parentheses around this "del" statement is also unnecessary (AND confusing):

del(colors["purple"])

This makes it look like "del" is a function, but it's not.

"del" is a statement:

del colors["purple"]

Watch out for parentheses that make STATEMENTS look like function calls.

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 That said, don't ALWAYS remove unnecessary parentheses.

For example, parens when making a tuple can sometimes improve readability.

I find this:
SENTENCE_ENDERS = ('.', '?', '!')

More readable than this:
SENTENCE_ENDERS = '.', '?', '!'

Consider readability when deciding how to use parentheses.
