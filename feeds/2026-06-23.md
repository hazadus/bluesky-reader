💬 Python Tip #174 (of 365):

Python modules are cached. 🧵

The first time each Python process imports a module, it caches it in sys.modules.

The second time you import the same module, Python returns the cached module object without re-running the module code.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3moxyfrfcyx2w)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 This means if you edit a file and then re-import it in the same REPL session, your changes won't appear.

Re-importing a module does not reload it.

The simplest fix: restart Python!

Though that's not the only way...

[Открыть пост](https://bsky.app/profile/trey.io/post/3moxyfrke5t2q)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 If you need to reload a module without restarting, you can use importlib.reload:

>>> import importlib
>>> importlib.reload(my_module)

But if you're not careful, using importlib.reload can result in two versions of your module's classes, class instances, etc.

Restarting the REPL is less messy.
