💬 Python Tip #141 (of 365):

Don't use bare "except" clauses 🧵

Instead of this:

try: ...
except: ...

Do this:

try: ...
except Exception: ...

Or this:

try: ...
except BaseException: ...

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mmfvgnvntk2y)

----
**🕒 02:33 👤 @trey.io (Trey Hunner)**
💬 If you need to catch EVERY exception either use:

1. "except Exception" (excludes system-exiting exceptions)
2. "except BaseException" carefully (catches system-exiting exceptions too)

Note that BaseException will catch the exception raised by the user hitting Ctrl+C as well as sys.exit calls.

💬 Python Tip #141 (of 365):

Be specific when handling exceptions 🧵

Unless you're trying to catch all possible exceptions (often for logging) I HIGHLY recommend being as specific as possible in exception handling.

Catch only the specific types you intend to handle.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mmhh5i7zz22m)

----
**🕒 17:22 👤 @trey.io (Trey Hunner)**
💬 Also prefer to catch specific exception types (e.g. ValueError) instead of broad ones (e.g. Exception).

And try to reason about the possible origins of a specific exception type before handling it.

Casting a wide net when exception handling can suppress real issues which can mask or create bugs!
