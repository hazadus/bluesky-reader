💬 Python Tip #143 (of 365):

Re-raise exceptions deliberately 🧵

When you need to re-raise an exception, you have 2 options:

1. Use a bare "raise" statement
2. Use "raise" with a "from" clause

#Python #DailyPythonTip

**🕒 05:49 👤 @trey.io (Trey Hunner)**
💬 If you put "raise SomeNewException()" in an except block, the traceback will indicate that an exception occurred while another exception was being handled.

If you instead write "raise SomeNewException() from e" then Python will indicate that the new exception is deliberately replacing the original.

**🕒 05:49 👤 @trey.io (Trey Hunner)**
💬 If you want to catch an exception, log it (or handle it in some other way), and then re-raise the same exception, use a bare "raise" statement.

If you use just "raise" (with nothing after it) in an "except" block, the original exception will be re-raised.
