**🕒 18:06 👤 @trey.io (Trey Hunner)**
💬 Python Tip #154 (of 365):

Writing a type that's "this or that"? Use the | operator instead of Union or Optional. 🧵

"int | str" reads better than "Union[int, str]", and "int | None" reads better than "Optional[int]".

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnfp6fml5y2x)

----
**🕒 18:06 👤 @trey.io (Trey Hunner)**
💬 The old way required imports:

from typing import Optional, Union
def parse(value: str) -> Optional[int]: ...
def normalize(value: Union[int, str]) -> str: ...

The newer way (3.10+), requires no imports at all:

def parse(value: str) -> int | None: ...
def normalize(value: int | str) -> str: ...

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnfp6fpg522m)

----
**🕒 18:06 👤 @trey.io (Trey Hunner)**
💬 Also note that "Optional[int]" means "int | None", NOT "this argument is optional".

A function argument CAN be "Optional[int]" without having a default value.

That's another reason I prefer "int | None" over "Optional[None]": it's harder to misunderstand the meaning.
