**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Python Tip #67 (of 365):

When making a class that should support inheritance, don't hard-code references to your own class. 🧵

If your class is meant to be inherited from, instead of hard-coding references to it, use type(self) to dynamically look up the current class.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mglg6skxrp24)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 This is particularly relevant in __repr__ methods.

Instead of this:

def __repr__(self):
    return f"MyClass(...)"

Do this:

def __repr__(self):
    class_name = type(self).__name__
    return f"{class_name}(...)"

This ensures instances of child classes show the right type in their repr()/str().
