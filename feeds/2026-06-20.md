Use b to set a breakpoint while you're in PDB 🧵

You can use the b command to set a breakpoint at a certain line of code:

(Pdb) b 4

💬 Python Tip #171 (of 365):

Customize PDB with a ~/.pdbrc file.  🧵

You can make a ~/.pdbrc file to add your own commands to the Python debugger.

This ISN'T a Python file, so it's a bit tricky to create, but spending a few minutes making one is well worth it.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3moqgxxyrnu2r)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 At startup, PDB runs commands from a ~/.pdbrc file.

The handiest thing to put there are custom PDB aliases.

Here's one that prints the local variables, each on its own line:

alias loc print(*(f"{name} = {value!r}" for name, value in vars().items() if not name.startswith("__")), sep="\n")

[Открыть пост](https://bsky.app/profile/trey.io/post/3moqgxy6bxb2j)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Type locals() in PDB and you'll see a dictionary:

(Pdb) locals()
{'self': , 'bonus': 10}

With our custom loc alias, each variable ends up on its own line:

(Pdb) loc
bonus = 10
self =

[Открыть пост](https://bsky.app/profile/trey.io/post/3moqgxyd55r2f)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Aliases can take arguments too. %1 is replaced by the first argument passed.

This vars alias prints an object's instance attributes:

alias vars print(*(f"%1.{k} = {v!r}" for k, v in vars(%1).items()), sep="\n")

Here's an example use:

(Pdb) vars self
self.‍name = 'Trey'
self.‍score = 42

[Открыть пост](https://bsky.app/profile/trey.io/post/3moqgxyhtvz2u)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Things to know about ~/.pdbrc files:

• It's not PDB, not Python... so each alias must be a single line
• You can chain Python statements with ;;
• Assigned variables leak into the surrounding scope, so some aliases end with a del to clean up

I share all 5 of my own PDB aliases in this blog post 👇
