💬 Python Tip #166 (of 365):

Consider using PDB to debug your code. 🧵

Instead of temporary print calls, consider using Python's breakpoint() function to pause your program and drop you into the Python debugger (PDB).

(This week's tips are all about debugging)

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3moeedyezhj2e)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Here's a tiny example:

from random import randint

breakpoint()

answer = randint(0, 1)
n = input("Guess: 0 or 1? ")

[Открыть пост](https://bsky.app/profile/trey.io/post/3moeedyhtcd24)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 When Python hits breakpoint(), you'll drop into PDB:

$ python guess.‍py
> /home/trey/guess.py(5)()
-> answer = randint(0, 1)
(Pdb) answer
1

Once you're in PDB, you can run Python code and step through your code.

**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 breakpoint() lets you pause your running program and introspect its state.

print tells you what happened. breakpoint() lets you pause while it's happening.
