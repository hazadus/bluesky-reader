**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ When an object is used in a boolean context (like in an "if" or "while" condition) it's implicitly converted to a boolean. This is called a "truth value" test, though colloquially it's pretty much always called "truthiness".

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mc3pb2hqh52c)

**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**
ğŸ’¬ For lists, strings, tuples, sets, dictionaries and pretty much all data structures (a.k.a. "collections"), a truthiness check is a non-emptiness check.

Meaning empty collections are falsey and non-empty ones are truthy.

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mc3pb2kj3a2c)

**ğŸ•’ 19:04 ğŸ‘¤ @trey.io (Trey Hunner)**

So for lists, this:

    ...

Is equivalent to this:

    ...

[ĞÑ‚ĞºÑ€Ñ‹Ñ‚ÑŒ Ğ¿Ğ¾ÑÑ‚](https://bsky.app/profile/trey.io/post/3mc3pb2mxgv2c)
