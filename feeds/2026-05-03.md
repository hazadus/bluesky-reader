**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #123 (of 365):

When accepting a file as input to your script, also accept piped input instead 🧵

Consider making a filename argument optional and reading sys.stdin if it isn't provided.

Or allow for the "-" convention, indicating "read from standard input".

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mkxqlrx3472r)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Reading from standard input allows users to pipe the output of another process into your script.

This common Unix pattern allows for piped input this:

$ some_preprocessing_script data.txt | python my_script.py

In addition to accepting a filename:

$ python my_script.py data.txt

[Открыть пост](https://bsky.app/profile/trey.io/post/3mkxqls2xrr2t)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 You could do this by making your input file argument optional, but sometimes that causes trouble, especially if your program accepts multiple arguments with one representing an input file and another representing an output file.

Instead, you could embrace the convention of specially handling "-".

[Открыть пост](https://bsky.app/profile/trey.io/post/3mkxqls56g72r)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 When "-" is specified as a filename, this program will read from standard input:

$ some_other_process | python my_script.py -

Sound tricky to implement?

It's probably not!

If you're using argparse or most other Python command-line argument parsing tools, this will work by default!
