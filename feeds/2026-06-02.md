**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #153 (of 365):

When you write type annotations, embrace duck typing: annotate with the most general type you can. 🧵

Don't require a list if any iterable will do. Don't require a dict if any mapping will do.

pymhttps://pym.dev/duck-typing/Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnd6mrdu4n27)

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Most functions that "accept a list" actually accept any iterable.

So instead of this:

def square_all(numbers: list[float]) -> list[float]:
    ...

Prefer the more permissive annotation:

from collections.‍abc import Iterable

def square_all(numbers: Iterable[float]) -> list[float]:
    ...

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnd6mrgmz42j)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 A rough hierarchy, from most general to most specific:

• Iterable: you'll only loop over it
• Sequence: you'll also index it, use len(), or loop more than once
• list: you'll also mutate it

Reach for the most general one that supports everything your code actually does.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnd6mriyot2f)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 The same idea applies to dictionaries.

If you only need to look up keys and loop, annotate with collections.‍abc.Mapping instead of dict. Only use MutableMapping when you need to change the object.

Annotating with behaviors instead of concrete types makes your functions usable by more callers.
