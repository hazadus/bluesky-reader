💬 Python Tip #169 (of 365):

Use PDB's post-mortem mode to debug exceptions 🧵

Python script raising an exception and want to drop into an interactive environment RIGHT after the exception occurs?

Use "python -m pdb your_script.py"

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3molg3bzg7o2g)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Using Python's "-m" flag will run the pdb module as a command-line program.

When launched like this, PDB enters "post-mortem debugging" mode.

You'll drop into PDB as soon as your program launches.

You'll want to type c to continue to the next breakpoint (yes, we haven't set one, stay with me).

[Открыть пост](https://bsky.app/profile/trey.io/post/3molg3cbn2t2o)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 In post-mortem debugging mode, whenever your script crashes, PDB will automatically stop at the point of the exception so you can inspect variables.

So you don't need to set a breakpoint. In post-mortem debugging mode, PDB will make a breakpoint right when an exception occurs.

[Открыть пост](https://bsky.app/profile/trey.io/post/3molg3celpp2s)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 It's pretty convenient to drop into PDB as soon as your program hits an uncaught exception.

The alternative is to put a try-except block in the right place in your code and then call breakpoint() in that block. That works, but it's as quick as putting "python -m pdb" before your script name.
