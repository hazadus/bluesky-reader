💬 Python Tip #162 (of 365):

Avoid modifying a list while you're looping over it. 🧵

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mo2iq3lkts2x)

----
**🕒 00:36 👤 @trey.io (Trey Hunner)**
💬 Adding or removing items while looping over a list can cause clear bugs due to skipped or double-visited items.

This loop misses -3:

>>> numbers = [4, -1, -3, 2, -1, 5]
>>> for n in numbers:
...     if n >> numbers
[4, -3, 2, 5]

[Открыть пост](https://bsky.app/profile/trey.io/post/3mo2iq3lvlk2x)

----
**🕒 00:36 👤 @trey.io (Trey Hunner)**
💬 I'd even avoid updating items in-place:

for i, n in enumerate(numbers):
    numbers[i] = n * 2

In-place updates are awkward to write. Also, if that list object was created somewhere else and passed to your code, mutating the original list object like that might upset the caller of your code.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mo2iq3lyjc2x)

----
**🕒 00:36 👤 @trey.io (Trey Hunner)**
💬 Instead of modifying a list in-place while looping, make a new list.

A list comprehension works great for this.

numbers = [n for n in numbers if n >= 0]

As a general rule: don't add, remove, or update items in a list while looping over it. Build a new list instead.
