**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #147 (of 365):

Sort by unbound method objects instead of using "lambda" 🧵

Say you're sorting mixed capitalization strings:

>>> frameworks = ["jQuery", "React", "Alpine", "htmx", "Svelte"]
>>> sorted(frameworks)
['Alpine', 'React', 'Svelte', 'htmx', 'jQuery']

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mmu3srsrjk2q)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 You could case-normalize the strings by supplying a key function that calls the casefold method on each string:

>>> sorted(frameworks, key=lambda s: s.casefold())
['Alpine', 'htmx', 'jQuery', 'React', 'Svelte']

But there's a better way...

[Открыть пост](https://bsky.app/profile/trey.io/post/3mmu3sry24z2i)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 We can do this:

>>> sorted(frameworks, key=str.casefold)
['Alpine', 'htmx', 'jQuery', 'React', 'Svelte']

Instead of creating our own function (with "lambda" or "def") that just delegates to the method on each of the given objects, we're just passing the unbound method object as the key function.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mmu3ss4wez2i)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Every method in Python is implemented as a function that lives on a class.

Most methods are meant to be called on class instances, so the unbound method object (the function on the class) accepts self as its first argument.

This trick works for ANY method that accepts 0 arguments (besides self).
