💬 Python Tip #125 (of 365):

Use function guards 🧵

Will your function need to exit early if a certain condition is met?

Could that condition be easily modeled as a precondition at the top of your function?

If so, you should use a function guard!

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3ml5pib54zi2o)

----
**🕒 03:00 👤 @trey.io (Trey Hunner)**
💬 Function guards occur before the main body of a function.

They check certain preconditions that indicate that the usual body of a function doesn't apply and then exit early by either returning or raising an exception.

Guards either either return early or raise an exception.

[Открыть пост](https://bsky.app/profile/trey.io/post/3ml5pib7wo52e)

**🕒 03:00 👤 @trey.io (Trey Hunner)**
💬 One flavor of a guard is an early return for a special case:

def get_files(path):
    if path.‍is_file():
        return [path]
    return path.rglob("*.py")

The other flavor is a precondition check that raises an exception...

**🕒 03:00 👤 @trey.io (Trey Hunner)**
💬 Like:

def gift_exchange(names):
    if len(names) < 2:
        raise ValueError("Need at least 2 participants")
    if len(names) != len(set(names)):
        raise ValueError("Names must be unique")
    # ...

Both flavors keep the main body of the function unindented and avoid deeply nested logic.
