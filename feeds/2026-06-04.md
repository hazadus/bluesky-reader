**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #155 (of 365):

When a method returns an instance of its own class, annotate the return type as Self. 🧵

This is the type-annotation version of preferring "type(self)" over hard-coding your class name.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mni7jc55zu24)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Hard-coded class names break for subclasses:

class Connection:
    def __enter__(self) -> "Connection":
        self.connect()
        return self

If PooledConnection inherits from Connection, a type checker will think "PooledConnection().__enter__()" returns a Connection, not a PooledConnection.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mni7jf6xtj2n)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 from typing import Self

class Connection:
    def __enter__(self) -> Self:
        self.connect()
        return self

Now a type checker knows that "PooledConnection().__enter__()" would return a PooledConnection.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mni7jfc3mn24)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Self (added in Python 3.11) is handy for alternative constructors, __enter__ methods, copy-style methods, and any "fluent" method that returns self.

It's the same lesson as "type(self)": don't hard-code references to your own class.
