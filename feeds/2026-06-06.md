**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Python Tip #157 (of 365):

Old type annotations still work, but newer Python has cleaner ways to write most of them. 🧵

Modernizing your annotations removes a lot of typing imports and boilerplate.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnnah4fmgn2b)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Instead of deprecated aliases:

from typing import Dict, List
def tally(words: List[str]) -> Dict[str, int]: ...

Use the built-in generics (they've been around since 3.9):

def tally(words: list[str]) -> dict[str, int]: ...

And use "X | Y" instead of Union and Optional (see tip 154).

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnnah7akgc2a)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 If you're on Python 3.12+, define type aliases with the type statement.

Old way:

from typing import TypeAlias
Vector: TypeAlias = list[float]

New way:

type Vector = list[float]

[Открыть пост](https://bsky.app/profile/trey.io/post/3mnnah7emmn23)

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Also for Python 3.12+, use the generic function/class syntax.

Old way:

from typing import TypeVar
T = TypeVar("T")
def first(items: list[T]) -> T: ...

New way:

def first[T](items: list[T]) -> T: ...
