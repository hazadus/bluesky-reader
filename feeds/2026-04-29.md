💬 Instead of argparse.FileType you should use pathlib.Path.

Using "type=Path" will auto-convert given arguments to pathlib.Path objects.

That way all command-line-provided file paths will turn into pathlib.Path objects automatically!

**🕒 23:00 👤 @trey.io (Trey Hunner)**
💬 Python Tip #119 (of 365):

Don't just rely on the built-in types for argparse's type parameter 🧵

You can create your own argparse "type" with a callable that accepts a string and returns the parsed object, raising a ValueError (with a user-facing message) if parsing failed.

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mko7bt3w2n2h)

----
**🕒 23:00 👤 @trey.io (Trey Hunner)**
💬 For example, if you wanted to accept a YYYY-MM-DD date for an argument, you could make a date function like this:

import datetime as dt

def date(string):
    return dt.‍date.strptime(string, "%Y-%m-%d")

And use "type=date" in your parser.add_argument call.

**🕒 23:00 👤 @trey.io (Trey Hunner)**
💬 Of course, in the specific example of parsing a date, you could specify "type=dt.‍date.fromisoformat" instead.

But then parsing errors would show fromisoformat to the user:

program: error: argument date: invalid fromisoformat value: '01/01/2025'

**🕒 23:00 👤 @trey.io (Trey Hunner)**
💬 To customize the error message, you can raise argparse.ArgumentTypeError instead of ValueError:

def date(string):
    try:
        return dt.‍date.fromisoformat(string)
    except ValueError:
        m = "invalid YYYY-MM-DD date: {string!r}"
        raise argparse.ArgumentTypeError(m) from None
