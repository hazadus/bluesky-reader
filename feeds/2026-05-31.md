💬 Python Tip #151 (of 365):

For truly unpredictable randomness, use the secrets module 🧵

Python's random module relies on pseudo-randomness, meaning it generates in a predictable order. If someone guesses the "seed" (it's based on the time) they'll know the numbers produced.

#Python #DailyPythonTip

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 For true randomness, use Python's secrets module instead.

The secrets module has some functions that are similar to those in the random module.

secrets.randbelow will generate a random integer (the upper bound is EXCLUSIVE, unlike random.randint):

>>> import secrets
>>> secrets.randbelow(10)
7

**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 Use secrets.choice to randomly select an item from a sequence:

>>> colors = ["blue", "purple", "green", "yellow"]
>>> secrets.choice(colors)
'purple'

Need a random module function that isn't directly available in secrets?

----
**🕒 18:04 👤 @trey.io (Trey Hunner)**
💬 SystemRandom includes methods equivalent to all random module functions:

>>> truerand = secrets.SystemRandom()
>>> truerand.randint(1, 5)
2
>>> truerand.shuffle(colors)

For numbers that SEEM random, random is good enough.

But for true randomness (e.g. generating a secure auth token) use secrets.
