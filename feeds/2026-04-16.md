**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Python Tip #106 (of 365):

Name ambiguous unicode characters with \N{...} 🧵

To represent a non-ASCII character, you can:
1. Copy-paste the character directly
2. Use \u or \U to represent the character by its code point
3. Use \N to represent the character by its name

#Python #DailyPythonTip

[Открыть пост](https://bsky.app/profile/trey.io/post/3mjniergqaw2k)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 Unicode characters copy-pasted into Python code look odd because fixed-width fonts often don't handle non-ASCII characters well.

Using a \u or \U prefix works:

message = "Yay! \u2728"

But unless you happen to know that \u2728 represents ✨, you won't know what that string actually looks like.

[Открыть пост](https://bsky.app/profile/trey.io/post/3mjnierk6nk2t)

----
**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 For readability's sake, I prefer to use "\N" to include Unicode characters within strings based on their name:

message = "Yay! \N{sparkles}"

The capitalization in the name doesn't matter:

message = "Yay! \N{SPARKLES}"

**🕒 22:45 👤 @trey.io (Trey Hunner)**
💬 If you don't know the name of a Unicode character, you could use unicodedata.‍name to look it up:

>>> import unicodedata
>>> unicodedata.‍name("\u2728")
'SPARKLES'
